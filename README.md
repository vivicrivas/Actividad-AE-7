# Actividad AE-7
## Descripción de la actividad
Esta actividad simula un flujo de trabajo colaborativo utilizando **Git** y **GitHub**
El objetivo principal es demostrar la capacidad de gestionar repositorios, manipular ramas, resolver conflictos de fusión y coordinar cambios mediante Pull Requests.

## Funcionalidades Implementadas
Durante el desarrollo se agregaron los siguientes componentes al archivo `index.html`:
* **Barra de Navegación (Navbar):** Un encabezado básico que simula la navegación principal del sitio.
* **Pie de Página (Footer):** Un elemento estructural para el cierre de la página.

## Estructura de Ramas
Para este examen se utilizó un modelo de ramificación basado en funcionalidades (feature branches):
* **`main`**: Rama principal que contiene el código estable y listo para producción.
* **`feature/navbar`**: Rama dedicada al desarrollo del encabezado, donde se practicó la resolución manual de conflictos.
* **`feature/footer`**: Rama utilizada para simular un flujo de trabajo remoto mediante la creación de un Pull Request en la interfaz de GitHub.
